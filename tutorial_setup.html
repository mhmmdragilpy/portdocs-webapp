🚀 Panduan Setup & Deployment Web App PortDocs

Ikuti panduan komprehensif ini untuk menyiapkan database (Supabase), publikasi online (Vercel), dan autentikasi Google. Tahapan ini disusun berurutan agar Anda tidak perlu bekerja dua kali.

Bagian 1: Persiapan Supabase Dasar

1. Buat Proyek Baru

Buka Supabase Dashboard, login, dan klik "New Project".

Beri nama proyek (misal: portdocs-app). Buat dan simpan Database Password Anda dengan aman.

2. Ambil Kredensial API

Pergi ke menu ⚙️ Project Settings -> API. Salin dua kunci berikut ke dalam file .env.local Anda:

  • Project URL → NEXT_PUBLIC_SUPABASE_URL
  • anon public → NEXT_PUBLIC_SUPABASE_ANON_KEY
  • service_role secret → SUPABASE_SERVICE_ROLE_KEY (Kunci rahasia untuk Admin)

3. Setup Storage (Bucket)

Pergi ke menu Storage. Klik New Bucket.

Beri nama draft-files. JANGAN centang "Public Bucket" agar dokumen KTP/Ijazah klien tetap rahasia. Klik Save.

4. Sinkronisasi Database via Drizzle ORM

Pergi ke menu ⚙️ Project Settings -> Database. Gulir ke bawah ke bagian Connection String (URI).

Salin URL tersebut dan masukkan ke variabel DATABASE_URL di file .env.local. Pastikan mengganti [YOUR-PASSWORD] dengan password database Anda sesungguhnya.

Lalu, di terminal VS Code komputer Anda, jalankan:

npm run db:push

Ini akan otomatis membuat tabel clients dan orders di Supabase Anda.

Bagian 2: Deployment ke Vercel (Online)

Kenapa Vercel Dulu? Kita butuh domain Vercel terlebih dahulu (contoh: portdocs.vercel.app) agar bisa didaftarkan di Google Cloud untuk keamanan *login* OAuth nantinya.

1. Push ke GitHub

Pastikan semua kode Anda telah di-*commit* dan di-*push* ke *repository* GitHub Anda.

2. Import Proyek di Vercel

Buka Vercel Dashboard dan login dengan akun GitHub Anda.

Klik "Add New..." -> Project. Pilih repository GitHub PortDocs Anda dan klik Import.

3. Mengatur Environment Variables

Sebelum menekan tombol "Deploy", buka menu tarik-turun Environment Variables di layar setup Vercel.

Tambahkan keempat variabel dari file .env.local Anda satu per satu ke dalam daftar:

  • NEXT_PUBLIC_SUPABASE_URL
  • NEXT_PUBLIC_SUPABASE_ANON_KEY
  • SUPABASE_SERVICE_ROLE_KEY
  • DATABASE_URL

4. Deploy!

Klik tombol Deploy. Vercel akan memberikan domain gratis (contoh: https://portdocs.vercel.app). Salin domain URL ini, kita akan membutuhkannya di langkah selanjutnya.

Bagian 3: Konfigurasi Google OAuth (Login Klien)

1. Tambahkan Domain Vercel ke Supabase

Buka Supabase Anda, pergi ke menu Authentication -> URL Configuration.

Pada bagian Site URL, masukkan domain Vercel yang baru Anda dapatkan (contoh: https://portdocs.vercel.app).

Pada bagian Redirect URLs, tambahkan https://portdocs.vercel.app/api/auth/callback. Klik Save.

2. Dapatkan Kredensial Google Cloud

Buka Google Cloud Console.

Buat kredensial OAuth 2.0 Client IDs.

Di bagian Authorized redirect URIs pada Google Console, masukkan URI Redirect dari Supabase (Biasanya berbentuk https://[PROJECT_ID].supabase.co/auth/v1/callback).

3. Aktifkan Google di Supabase

Pergi ke menu Authentication -> Providers di Supabase.

Pilih Google dan aktifkan. Masukkan Client ID dan Client Secret yang baru Anda dapatkan dari Google Cloud. Klik Save.

🎉 Selesai! Web App Anda sekarang sudah online, dilengkapi dengan penyimpanan *database* yang aman dan fitur login menggunakan Google.
